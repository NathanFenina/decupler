Le principe : un cerveau, plusieurs bras

Claude Code est le cerveau. Les MCP (Model Context Protocol) sont les bras : chacun connecte Claude à un outil réel (ton navigateur, Search Console, un outil de mots-clés, ton WordPress). Une fois branchés, Claude ne « raconte » plus le SEO — il le fait : il lit tes vraies données, rédige, visualise le rendu, publie.

  • MCP Chrome DevTools — Claude ouvre un navigateur et voit le rendu HTML avant publication
  • MCP Search Console (via Windsor.ai) — les vraies impressions, positions, clics de ton site
  • MCP mots-clés (Ubersuggest / Ahrefs) — volumes, difficulté, intention
  • MCP imagegen — génère les visuels des articles
  • Publication WordPress — via l'API REST (mot de passe d'application)
💡

Deux familles de MCP : ceux qui tournent en local (lancés par npx, déclarés dans un fichier de config) et les connecteurs claude.ai (branchés en un clic depuis les réglages). On voit les deux.

Étape 0

Installer Claude Code

Si ce n'est pas déjà fait, installe Claude Code (Node.js requis), puis lance-le dans le dossier de ton projet.

terminal
# Installation (une fois)
npm install -g @anthropic-ai/claude-code
# Lancer dans ton projet
cd mon-projet
claude

Les MCP se déclarent soit dans un fichier .mcp.json à la racine du projet, soit via la commande claude mcp add. On utilise les deux ci-dessous.

Étape 1 · Voir le rendu

MCP Chrome DevTools

Le plus important pour la qualité : Claude ouvre un vrai navigateur, charge la page et en lit le rendu (screenshot + arbre d'accessibilité). Fini le « publié à l'aveugle » — on vérifie le HTML avant la mise en ligne.

.mcp.json
{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": ["-y", "chrome-devtools-mcp@latest", "--viewport", "1440x900", "--isolated"]
    }
  }
}

Équivalent en une commande :

terminal
claude mcp add chrome-devtools -- npx -y chrome-devtools-mcp@latest --isolated
✅

Vérifier : dans Claude, demande « ouvre cette page et fais un screenshot » — s'il te renvoie une capture, c'est branché.

Étape 2 · Générer les visuels

MCP imagegen

Pour que chaque article sorte avec ses images, on branche un générateur (ici gpt-image-1 d'OpenAI). Il faut une clé API OpenAI.

.mcp.json
{
  "mcpServers": {
    "imagegen": {
      "command": "npx",
      "args": ["-y", "imagegen-mcp", "--models", "gpt-image-1"],
      "env": { "OPENAI_API_KEY": "sk-…ta-clé…" }
    }
  }
}
🔒

Ne commit jamais ta clé. Garde-la en variable d'environnement ou dans un fichier ignoré par git.

Étape 3 · Les vraies données

Search Console via Windsor.ai

Pour piloter avec des faits (impressions, positions, clics), on branche Google Search Console. Le plus simple : le connecteur Windsor.ai, qui expose GSC (et 300+ sources) en MCP côté claude.ai.

📌 Connexion (côté claude.ai)

  1. Va dans Réglages → Connecteurs (claude.ai/customize/connectors)
  2. Ajoute Windsor.ai et autorise-le
  3. Dans Windsor, connecte la source Search Console et sélectionne la propriété de ton site
  4. Dans Claude, demande les requêtes des 90 derniers jours (champs : query, clicks, impressions, ctr, position, page)

Claude récupère alors les données brutes, que tu normalises en une commande pour repérer les quick wins (pages en position 5-20 qui peuvent passer en page 1) :

terminal
python3 scripts/gsc_normalize.py \
  --in .cache/gsc-raw.json \
  --out .cache/gsc.json
Étape 4 · Les mots-clés

Ubersuggest ou Ahrefs

Pour les volumes, la difficulté et l'intention, on branche un outil de mots-clés en MCP. Ubersuggest et Ahrefs exposent tous deux un connecteur — même logique de branchement que Windsor.

📌 Branchement

  1. Ajoute le connecteur (Ubersuggest ou Ahrefs) dans Réglages → Connecteurs, ou déclare son serveur MCP dans .mcp.json s'il tourne en local
  2. Vérifie l'authentification (compte + tier) avant de lancer
  3. Demande à Claude : suggestions de mots-clés, volumes et difficulté sur tes thématiques (marché FR)
🎯

Objectif : pas 500 mots-clés dans un tableur, mais une liste courte priorisée par intention commerciale et faisabilité. (La transformation en plan éditorial complet fait l'objet d'un autre guide.)

Étape 5 · Publier

Publication WordPress (API REST)

On publie directement dans WordPress via son API REST, avec un mot de passe d'application (révocable, sans exposer ton vrai mot de passe). Crée-le dans WordPress : Utilisateurs → Profil → Mots de passe d'application.

.env
# Jamais commité
WP_SITE_URL=https://ton-site.com
WP_USER=ton-utilisateur
WP_APP_PASSWORD=xxxx xxxx xxxx xxxx xxxx xxxx

Un petit script POST le contenu (article ou page), par défaut en brouillon pour relecture :

terminal
python3 scripts/wp_publish.py --type post \
  --title "Mon titre" --slug mon-titre \
  --content-file article.html --status draft
🖼️

Les images montent aussi via l'API (/wp-json/wp/v2/media) — Claude les génère, les compresse, les téléverse, puis insère l'URL dans le contenu.

Étape 6 · Le savoir-faire

Les Skills : figer tes méthodes

Un Skill, c'est une méthode écrite une fois que Claude applique à l'identique à chaque fois (rôle + règles + format). C'est ce qui rend la production cohérente et scalable. Chaque étape du pipeline a le sien : découverte de mots-clés, rédaction experte, lead magnet…

~/.claude/skills/redaction-expert/SKILL.md
---
name: redaction-expert
description: Rédige un article SEO+GEO profond et sourcé.
---
# Règles
1. Recherche AVANT d'écrire (données, SERP, sources)
2. Chaque paragraphe = une donnée / un exemple / une méthode
3. Descendre au "comment" technique, pas au "quoi"
4. Publier en brouillon, jamais inventer de chiffres

En Claude Code, un dossier ~/.claude/skills/<nom>/SKILL.md est auto-découvert. Tu invoques ensuite le skill par son nom.

Étape 7 · Suivre

Suivi & reporting

La boucle se referme avec le suivi. Le MCP Search Console alimente en continu un tableau de bord : ce qui monte, ce qui stagne, où réinvestir. On regénère un rapport mensuel automatiquement, et on retrace les positions des mots-clés via le MCP mots-clés.

  • Positions & impressions : GSC en temps réel (Windsor)
  • Suivi des mots-clés cibles : Ubersuggest / Ahrefs
  • Rapport mensuel : généré et mis en forme automatiquement
La boucle

Comment tout s'enchaîne

Mots-clés→ Rédaction (skill)→ Rendu (Chrome)→ Images (imagegen)→ Publication (WP)→ Suivi (GSC)

Chaque brique alimente la suivante : les données GSC révèlent les quick wins, le mot-clé nourrit la rédaction experte, Claude visualise le rendu avant de publier, WordPress reçoit l'article maillé et illustré, et GSC mesure le résultat — qui relance le cycle. Un seul cerveau, une seule boucle.

FAQ

Questions fréquentes

Faut-il savoir coder ?

Non, mais être à l'aise avec un terminal aide. Les MCP se branchent en copier-coller (un fichier .mcp.json ou quelques commandes) et les connecteurs claude.ai se branchent en un clic. Le reste, c'est Claude qui l'exécute.

C'est quoi exactement un MCP ?

Le Model Context Protocol est un standard qui connecte un modèle (Claude) à des outils externes. Chaque MCP donne à Claude une capacité réelle : ouvrir un navigateur, lire Search Console, publier sur WordPress, générer une image.

Ça marche avec Ahrefs au lieu d'Ubersuggest ?

Oui. La logique est identique : tu branches le connecteur mots-clés de ton choix. Ubersuggest et Ahrefs exposent tous deux un accès exploitable en MCP.

Et si je ne suis pas sur WordPress ?

La publication passe par une API : WordPress via son API REST aujourd'hui, la même logique s'applique aux autres CMS qui exposent une API ou un MCP dédié.
